# 데이터 수집 가이드

**작성일**: 2025-10-15

## 📊 데이터 수집 원칙

### 1. 공식 출처 우선
- ✅ 브랜드 공식 웹사이트
- ✅ 식품안전나라 (식품의약품안전처)
- ✅ 한국소비자원 공식 보고서
- ✅ USDA FoodData Central
- ⚠️ 필라이즈 등 제3자 앱 (참고만, 공식 출처 필요)
- ❌ 개인 블로그, 커뮤니티 추정치

### 2. 데이터 품질 등급

#### A등급: 공식 데이터
- 브랜드 공식 영양성분표
- 정부 기관 공식 데이터
- 검증된 실측 데이터
- **현재**: 3개 메뉴

#### B등급: 부분 추정 포함
- 공식 칼로리 + 성분표 추정
- 유사 제품 기반 추정
- 계산 가능한 추정치
- **현재**: 4개 메뉴

#### C등급: 전체 추정 (미사용)
- 완전 추정치
- 평균값 기반
- **사용하지 않음**

### 3. 필수 검증 항목
1. ✅ 메뉴명이 실제 존재하는 메뉴인가?
2. ✅ 이미지가 실제 제품과 일치하는가?
3. ✅ 중량이 합리적인가? (일반적으로 800-1200g/마리)
4. ✅ 칼로리가 합리적인가? (100g당 200-300kcal 범위)
5. ✅ 출처가 명확한가?

## 📋 데이터 수집 프로세스

### 1단계: 메뉴 확인
```bash
# 브랜드 공식 사이트에서 메뉴 확인
curl -s "https://브랜드사이트/menu" | grep "메뉴명"
```

**체크포인트**:
- [ ] 메뉴명이 실제로 존재하는가?
- [ ] 뼈/순살 구분이 명확한가?
- [ ] 중량 정보가 있는가?

### 2단계: 이미지 확인
```bash
# 이미지 URL 추출
curl -s "https://브랜드사이트/menu" | grep -oP 'src="[^"]*메뉴명[^"]*\.(jpg|png)"'
```

**체크포인트**:
- [ ] 이미지가 실제 제품 사진인가?
- [ ] 메뉴명과 이미지가 일치하는가?
- [ ] HTML 전후 컨텍스트 확인했는가?

### 3단계: 영양 데이터 수집

#### 우선순위 1: 브랜드 공식 영양성분표
```bash
# 공식 사이트에서 영양성분표 확인
curl -s "https://브랜드사이트/nutrition" | grep "영양"
```

#### 우선순위 2: 정부 기관 데이터
- 식품안전나라: https://www.foodsafetykorea.go.kr/portal/healthyfoodlife/foodnutrient/simpleSearch.do
- 한국소비자원 보고서

#### 우선순위 3: 계산 및 추정
- 기준 메뉴의 영양소 비율 활용
- 추가 소스/토핑의 영양소 합산
- **반드시 출처에 추정 방법 명시**

### 4단계: 데이터 검증
```typescript
// mockData.ts에 추가 후 검증
const newMenu = {
  name: '메뉴명',
  calories: 1000, // 100g당 200-300kcal 범위인지 확인
  // 3대 영양소 합계 검증: (protein * 4) + (carbs * 4) + (fat * 9) ≈ calories
  protein: 50,  // 4kcal/g
  carbs: 80,    // 4kcal/g
  fat: 60,      // 9kcal/g
  // 계산: 50*4 + 80*4 + 60*9 = 200 + 320 + 540 = 1060 kcal (합리적)
};
```

**검증 체크리스트**:
- [ ] 칼로리 범위 합리적 (100g당 200-300kcal)
- [ ] 3대 영양소 합계와 칼로리 일치 (±10%)
- [ ] 나트륨 범위 합리적 (1000-5000mg)
- [ ] 중량이 합리적 (300g-1200g)
